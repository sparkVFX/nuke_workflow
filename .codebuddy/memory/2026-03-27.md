# 2026-03-27 工作日志

## 修改 Prompt/Record 节点 UI 布局

**需求**：用户希望 NanoBanana Prompt 节点和 VEO Record 节点的 Properties 面板分为两部分：
- **上半部分（只读）**：显示原始生成记录（Model、Ratio、Resolution、Seed、Prompt、Negative Prompt 等），放在一个带边框的 QFrame 中
- **下半部分（可编辑）**：和生成器节点一样的完整参数控件（Model 下拉框、Ratio、Resolution、Seed、Prompt 编辑框、Negative Prompt 编辑框），以及 REGENERATE 按钮，用户可以修改参数后重新生成，替换当前结果

**修改的文件**：
1. `ai_workflow/nanobanana.py` — 重写 `NanoBananaPromptWidget._build_ui()` 和 `_load_from_node()`，恢复 `_regenerate()` 和 `_toggle_ui()` 功能
2. `ai_workflow/veo.py` — 重写 `VeoRecordWidget._build_ui()` 和 `_load_from_node()`，恢复 `_regenerate()` 和 `_toggle_ui()` 功能

**关键改动**：
- 之前的版本把整个面板改成了只读（包括 prompt 和参数），但用户只需要历史记录部分只读
- 现在上半部分用 QFrame 包裹只读信息，下半部分是完整可编辑的参数 + 重新生成按钮
- 重新生成会使用缓存的输入图片（从 node knob 中读取 JSON），调用相同的 Worker 线程
- 完成后调用 `update_prompt_read_node()` 或 `update_record_read_node()` 替换当前 Read 节点的文件路径

## VEO 视频输出转码 MP4 → ProRes MOV

**需求**：VEO 生成的视频是 MP4 格式，用户需要转为 ProRes 编码的 MOV 格式以便 Nuke Read 节点更好地读取

**修改**：`ai_workflow/veo.py`
- 新增 `_find_ffmpeg()` 函数：按优先级搜索 ffmpeg（Nuke 目录 → 系统 PATH → 常见 Windows 路径）
- 新增 `_convert_mp4_to_prores()` 函数：使用 ffmpeg 将 MP4 转为 ProRes 422 MOV（prores_ks 编码器，profile 2），转码成功后自动删除中间 MP4 文件
- 在 `VeoWorker.run()` 中，视频下载保存为 mp4 后自动调用转码，最终传给 Read 节点的是 `.mov` 路径
- 首次生成和重新生成共用同一个 VeoWorker，因此都会自动转码
- 如果 ffmpeg 不可用，会回退到使用原始 MP4（不会报错阻断流程）
