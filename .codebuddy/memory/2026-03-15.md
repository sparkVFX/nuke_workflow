# 2026-03-15 工作记录

## google-genai 安装方案（Nuke 内置 Python 自动安装）

- ~~离线安装方案已废弃~~（因 pydantic_core 等 C 扩展包的 wheel 版本必须与 Nuke 内置 Python 精确匹配）
- ~~install.bat pip install 方案也已废弃~~（系统 Python 版本可能与 Nuke 内置 Python 不同，导致 pydantic_core._pydantic_core 加载失败）
- **当前方案**：在 Nuke 启动时（`init.py`），用 **pip 的 Python API**（`pip._internal.cli.main`）在 Nuke 进程内直接安装，确保编译包版本与 Nuke Python 版本精确匹配
- 安装流程：
  1. `install.bat` 只负责配置文件（init.py、menu.py、conf）和部署 studio_listener，**不安装 pip 依赖**
  2. Nuke 启动时 `init.py` 检测 `site-packages/` 是否存在且 `pydantic_core._pydantic_core` 能否加载
  3. 如果版本不匹配或不存在：先 `ensurepip.bootstrap()` 确保 pip 可用，再用 `pip._internal.cli.main` API 安装
  4. 如果 pip API 失败，回退到 `subprocess.run([sys.executable, "-m", "pip", ...])` 方式
- `.nuke/init.py` 注入的代码也包含同样的检测和自动安装逻辑
- `install.bat` 最终提示信息改为英文，避免 bat echo 中文被解析为命令的问题
- `uninstall.bat`：卸载时清理 `site-packages/` 目录

### 修复的 bug
1. `.nuke/init.py` 有旧的无标记代码（第1-25行）未被 install.bat 清理 → 已直接覆盖为仅含标记块的干净版本
2. `install.bat` echo 输出含中文导致 `'时会自动安装' is not recognized` 和 `'uke' is not recognized` 错误 → 改为英文提示
3. `sys.executable` 在 Nuke 中指向 Nuke.exe 而非 python.exe，subprocess 方式不可靠 → 改为优先使用 pip 的 Python API（`from pip._internal.cli.main import main`）直接在进程内安装
4. Nuke 可能不自带 pip → 增加 `ensurepip.bootstrap()` 自动引导

## studio_listener.py 部署方式变更

- **旧方式**：在 `init.py` 中通过 `import ai_workflow.studio_listener` 手动加载（已移除）
- **新方式**：`install.bat` 安装时将 `studio_listener.py` 拷贝到 `.nuke/Python/Startup/` 目录，NukeStudio 自动扫描并执行
- `install.bat`：创建 `.nuke/Python/Startup/` 目录，`copy /y` 拷贝文件
- `uninstall.bat`：清理 `.nuke/Python/Startup/studio_listener.py`
- `init.py`：移除旧的 hiero import 代码，添加注释说明新方式

## veo.py 功能改进

### FirstFrame 模式时长选择
- **修改前**：1080P 分辨率下所有模式都强制 8 秒
- **修改后**：FirstFrame 和 Frames 模式不再受 1080P 强制 8 秒限制，用户可自由选择 4/6/8 秒
- 只有 Ingredients 模式仍然强制 8 秒（API 要求）

### 缓存图片命名
- **修改前**：`{gen_name}_{ref_label}_{nuke.frame()}.png`，基于 Nuke 时间线帧号
- **修改后**：`{gen_name}_{ref_label}_frame{idx+1}.png`，使用简单递增序号 frame1/frame2/frame3
- 不再依赖 Nuke 当前时间帧位置
